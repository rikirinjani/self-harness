Benchmark: BENCH-C-02
Title: Bug Fix with Regression Check

BUG DESCRIPTION:
The update_summary() function in benchmarks/run.py hardcoded total_benchmarks=20
instead of counting actual result files. This caused:
- summary.json showed total_benchmarks=20 when only 15 results existed
- Pass rate was calculated as 15/20 = 75% instead of 15/15 = 100%

FIX APPLIED (benchmarks/run.py line 258):
- Before: "total_benchmarks": 20,
- After:  "total_benchmarks": total,

ADDITIONAL FIXES DISCOVERED DURING REGRESSION CHECK:
1. open() calls in update_summary() and print_summary() lacked encoding="utf-8"
   → caused UnicodeDecodeError when result JSONs contained non-ASCII text
2. Fixed cat_letter → category mapping (split on "-" gave letter, not full name)

TEST (benchmarks/outputs/test_C02_bug.py):
- Asserts total_benchmarks == actual result file count
- Before fix: AssertionError — total_benchmarks=20, actual=16
- After fix: PASS — total_benchmarks=16, completed=16

VERIFICATION:
- python run.py --summary shows total_benchmarks now matching completed count
- All existing functionality preserved
