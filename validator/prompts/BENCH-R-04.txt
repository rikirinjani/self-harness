## Cluster 2: Scope Overstep / Missing Guardrails (8 failures)

**Root Cause:** Agents treat exploratory or investigative instructions as implementation instructions when the task boundary is not explicitly restricted. A "investigate X" prompt gets interpreted as "build X" because the agent defaults to action-oriented behavior.

**Surface Symptoms:** Building dashboards, running experiments, constructing sectors — all things the agent was asked to investigate, not implement.

**Supporting Traces:**
- `2026-06-29T11-29-27-712Z-workflow-built-sectors-directly.json`
- `2026-06-30T09-36-15Z-workflow-scope-overstep.json`
- `2026-06-30T10-34-37Z-workflow-heatmap-overstep.json`
- `2026-06-30T11-33-51-731Z-workflow-built-dashboard.json`
- `2026-07-01-workflow-memory-pin.json`
- `2026-06-29T18-53-12-214Z-workflow-missing-git-push.json`
- `2026-06-30T01-33-19-874Z-workflow-wrong-handoff-status.json`
- `2026-07-01T11-16-00-workflow-figures-not-embedded.json`

**Weakness Description:** Agents lack a "read-only by default" instinct. When instructions are ambiguous ("analyze the project structure, look into X"), the agent defaults to modifying files rather than producing analysis. The distinction between "research" and "implement" must be explicit in every task prompt.

## Cluster 3: External Dependency / Environment Failure (7 failures)

**Root Cause:** Agent proceeds without verifying that external dependencies (hardware, API keys, tool versions, network) are available. Failure is discovered at runtime, not pre-flight.

**Surface Symptoms:** Kaggle training iterations, FAISS timeout on CPU, encoding errors, missing imports at runtime.

**Supporting Traces:**
- `2026-07-04-kaggle-training-iterations.json`
- `20260708-tool-deployed-to-hf-space-but-faiss.json`
- `20260707-hf-search-encoding-issues.json`
- `2026-07-04-process-batch-failures.json`
- `20260707-memory-search-other-session.json`
- `20260708-config-added-import-but-didn't-update.json`
- `2026-06-30-config-economy-wars-init.json`

**Weakness Description:** No pre-flight check step exists in the agent workflow. The agent should verify (a) environment availability, (b) dependency versions, (c) resource constraints before executing a plan. This is a workflow gap, not a prompt gap.

## Cluster 4: Reasoning / Algorithm Edge Cases (5 failures)

**Root Cause:** Implementation logic handles the happy path but misses edge cases (empty input, encoding issues, format assumptions). Not caught by tests because tests also follow the happy path.

**Surface Symptoms:** Idle loop with no termination, seed derivation returning NaN, JSON corruption from regex patch.

**Supporting Traces:**
- `2026-06-30T06-00-00Z-reasoning-idle-loop.json`
- `2026-06-30T10-30-53Z-reasoning-seed-derivation-bug.json`
- `2026-06-29-corruption-json-regex-patch.json`
- `2026-07-01-config-pdf-read-fallback.json`
- `2026-07-03T091823-custom-domain-cookie.json`

**Weakness Description:** No systematic edge-case thinking in implementation. A "doubt-driven development" step (adversarial review before committing logic) would catch these before they reach production. This maps to Phase 3 (Validator).

---

## Cluster 5: Cross-Session Context Isolation (4 failures)

**Root Cause:** Each session starts with no memory of other sessions' work. Duplicates, wrong repo references, and lost entities occur because nothing persists between sessions.

**Surface Symptoms:** Same entry added twice, wrong GitHub repo cloned, entities invisible across sessions.

**Supporting Traces:**
- `2026-06-30-communication-nonstop-execution.json`
- `2026-07-01-communication-netflix-duplicate.json`
- `20260708-wrong-hermes-repo.json`
- `20260704-fix-invalid-repo-url.json`
- `20260707-memory-search-other-session.json`

**Weakness Description:** No shared state between sessions. Each agent acts as if it is the first and only agent. A session-state registry or shared project map would prevent duplicates and wrong-repo mistakes.
